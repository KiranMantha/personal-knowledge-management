In the dynamic field of cloud computing services, the providers that most commonly stand out are: Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform (GCP).

There might be several categories for modern Cloud Services, such as Computing Instances, Serverless, Load Balancing, Storage, and much more.

In many cases, a cloud service present in one of these providers has an equivalent in the others. The goal of this post is to help you navigate this myriad of different Cloud Services and find their equivalent in the other providers.

## Categories

### Cloud Computing

Press enter or click to view image in full size

![]()

This category includes services for the creation of customizable virtual servers, where you are able to choose the available memory, CPU, and storage, as well as to define network and security configuration.

You are also going to find services where you can just deploy an application without needing to worry about its underlying infrastructure. This is the case of Elastic Beanstalk (AWS) or App Engine (Google Cloud).

### Storage

Press enter or click to view image in full size

![]()

Cloud providers are able to offer a range of storage services that meet different needs. Normally, virtual server storage is ephemeral, which means that whenever an instance gets killed, everything stored in the filesystem gets lost. The Block Storage options can be attached to virtual servers so that you can have data persistency.

Cloud Storage options serve for multi-purpose object storage, which could be used for serving multimedia content, or just as a backup. In addition, if you are dealing with objects that are not going to be accessed often, you can use specific services that will take care of that at a better cost.t.

### Containers and Serverless

Press enter or click to view image in full size

![]()

The popularity of serverless computing has increased for its ease of use and scalability. There you have the opportunity to deploy services without dealing with any infrastructure. Function-as-a-service (FaaS) options allow build event-driven applications at scale, where you are billed for only the compute resources you utilize for handling requests.

Options such as Container Apps (Azure) and Cloud Run (Google Cloud) are designed to simplify the deployment of containerized services and have the scaling managed for you.

You can also choose to go with a fully managed Kubernetes service, which simplifies the tasks of building and operating Kubernetes Clusters.

### Databases

![]()

A reliable database service is also essential to keep modern applications running in a cloud environment. There are different solutions that suits different needs, ranging from relational databases to different kinds of NoSQL options.
